    スポットライト：Linux Foundation のセキュリティに関する取り組み

    Ashwin Ramaswami | 2024年2月5日

    Linux Foundationのすべてのプロジェクトコミュニティにおいて、セキュリティは引き続き注力分野となっています。昨年1年間のコミュニティの活動について詳しく紹介するとともに、2024年以降に取り組むべき課題について考えてみたいと思います。

    本日は、Linux Foundationプロジェクトコミュニティのうち、Open Source Security Foundation（OpenSSF）、FINOS Common Cloud Controls Project、OpenChain、SPDXの4つの最近の取り組みと成果に焦点を当てます。各コミュニティは、それぞれ異なる観点からセキュリティの側面に取り組んでおり、すべての人にとってより安全なソフトウェアエコシステムを持つという共通の目標に貢献しています。

    OpenSSF

    Open Source Security Foundation (OpenSSF) は、私たちすべてが依存しているオープンソースソフトウェア (OSS) の開発、メンテナンス、および消費を、持続可能な方法でより安全に行うことを目指す、Linux Foundation に属する組織です。これには、コラボレーションの促進、ベストプラクティスの確立、革新的なソリューションの開発が含まれます。

    OpenSSFは、さまざまな業界の100を超えるメンバーを擁しています。さらに、OpenSSFには、10のワーキンググループがあり、その中には、オープンソース開発者のためのベストプラクティスに関するワーキンググループや、サプライチェーンの整合性に関するワーキンググループなどがあります。OpenSSF内のその他のイニシアチブとしては、ビルドアーティファクトとコンポーネントのセキュリティ保護と検証に使用されるプロジェクトであるSigstoreや、資金提供とツーリングを通じてOSSメンテナーと提携し、未発見の脆弱性を体系的に発見して修正するイニシアチブであるAlpha-Omegaなどのプロジェクトがあります。

    2023 年、OpenSSF は、fundamentals of developing secure software のコースに 22,000 人を超えるソフトウェア開発者を登録しました。また、Concise Guide for Developing More Secure Software、Concise Guide for Evaluating Open Source Software、Compiler Options Hardening Guide for C and C++ など、コミュニティを支援するための新しいガイドも開始しました。OpenSSF Best Practices Badge program, which provides a larger checklist of best practices, には、現在 6,400 件を超えるプロジェクトが参加しています。人工知能/機械学習 (AI/ML) セキュリティと、多様性、公平性、包括性 (DEI) に関する新しいワーキンググループなど、2023 年には新しい取り組みが開始されました。

    OpenSSFは、9月に開催されたセキュアオープンソースソフトウェアサミットのように、官民連携で重要なインフラストラクチャの保護に取り組んでいます。また、米国ホワイトハウスの国家サイバーディレクター室（ONCD）とそのパートナーであるオープンソースソフトウェアセキュリティイニシアチブ（OS3I）からの、オープンソースソフトウェア（OSS）のセキュリティとメモリ安全プログラミング言語に関する情報提供要請（RFI）への回答もその1つです。また、OpenSSFは、ソフトウェアのセキュリティを向上させるために「AIとサイバーセキュリティの最高の人材」を集めることを目的とした、DARPA AIサイバーチャレンジ（AIxCC）の研究チャレンジアドバイザーも務めています。OpenSSFはまた、カナダ、スペイン、日本でOpenSSF Daysを開催するなど、世界中で参加者を増やす取り組みも行いました。詳細については、2023 OpenSSF年次レポートをご覧ください。

    FINOS Common Cloud Controlsプロジェクト

    FINOS Common Cloud Controls (FINOS CCC) は、金融サービス業界における準拠したパブリッククラウド展開のために一貫したコントロールを記述するように設計されたオープンスタンダードプロジェクトです。この標準は、主要なクラウドサービスプロバイダー全体の共通サービスに対する、サイバーセキュリティ、回復力、コンプライアンスの各コントロールの統一されたセットを目指しています。サービスと脅威の統一された分類法を作成することで、クラウドの集中によるシステムリスクの軽減にも役立ちます。Citi によって最初に提案された FINOS CCC は、2023 年 7 月に Fintech Open Source Foundation である 結成されました。2023 年 10 月に、プロジェクトはオープンソース化され、現在正式に参加と貢献が可能になっています。このプロジェクトは現在、GitHub で公開されており、誰でも参加および貢献できます。

    2023年11月1日、ニューヨークで開催されたOpen Source in Finance Forum（OSFF）で、FINOS CCCが紹介されました。CitiのCTOであるJim Adams氏が主導し、共通サービス分類、脅威を認識したコントロール、Open Security Controls Assessment Language（OSCAL）を使用した自動評価を含むFINOSのアプローチを紹介しました。OSCALは、セキュリティ制御関連情報を表現するための機械可読なフレームワークであり、業界およびNISTと共同で開発されています。FINOS CCCおよび関連するイニシアチブの目標は、金融サービス業界のニーズに対応するために、業界と標準化団体を結びつける協調的なエコシステムを構築することです。

    OpenChain プロジェクト

    OpenChainは、高品質なオープンソースライセンスコンプライアンスプログラムとオープンソースセキュリティ保証プログラムの主要な要件に関する国際標準を開発および維持するLinux Foundationプロジェクトです。OpenChain プロジェクトには、ソフトウェアサプライチェーンのセキュリティと回復力を向上させるために活動する、1,000社を超える企業の広範なグローバルコミュニティがあります。

    2023 年 12 月、OpenChain Project のISO/IEC 18974:2023 が正式な ISO 規格として公開されました。ISO/IEC 18974:2023 は、以前は OpenChain セキュリティ保証仕様 1.1 として知られており、質の高いオープンソースセキュリティ保証プログラムの主要な要件を定義する、デファクトスタンダードの業界標準です。この規格を採用することで、組織は、CVE、GitHub 依存関係アラート、パッケージマネージャーアラートなどのセキュリティ脆弱性の問題について、オープンソースソフトウェアをより適切にチェックできます。詳細については、https://www.openchainproject.org/security-assurance をご覧ください。

    SPDXプロジェクト

    最後に、SPDX プロジェクトの取り組みを強調したいと思います。SPDX は、ソフトウェアを構成するコンポーネントのリストである SBOM（Software Bill of Materials）を記述するオープンスタンダードです。共通フォーマットとして、SPDX は重要なリリースデータの共有に関連する冗長な作業を削減し、配布とコンプライアンスを効率化します。SPDX 仕様は、国際的なオープンスタンダード（ISO/IEC 5692:2021）です。

    2023 年には、SPDX 3.0 のリリースに向けて作業が続けられました。リリース候補は 5 月に発表されました。この新しいバージョンは、新しいセキュリティユースケースに SPDX 標準を拡張するのに役立つだけでなく、AI およびデータ来歴のトレーサビリティも向上させます。焦点は、ソフトウェアエンジニア、セキュリティ専門家、データサイエンティスト、法務およびコンプライアンス専門家が簡単にオンボーディングして利用できるようにすることです。SPDX コミュニティは、GitHub での SPDX 3.0 モデルへのフィードバックと貢献を引き続き受け付けています。

    2024年に向けて

    最後に、Linux Foundationは、すべてのプロジェクトにおいてセキュリティの維持に取り組んでいます。Linux Foundationがホストするプロジェクトの脆弱性開示ポリシーはこのページに掲載されており、そこで、私たちの活動において脆弱性を発見した場合にどのように報告すべきかについて説明されています。

    2024 年も、サイバーセキュリティに関するこれらすべての取り組みを基盤として、すべての人にとってより優れたエコシステムを確保していきたいと考えています。私たちは、セキュリティの最先端技術を進歩させることに焦点を当てたイベントを引き続き開催します。たとえば、2024 年 6 月 26 日から 27 日にワシントン州シアトルで開催されるCloudNativeSecurityCon North Americaや、今後の OpenSSF Days などです。ソフトウェアセキュリティに関する最新のトレンドとベストプラクティスを常に把握しておくために、LF トレーニングと認定が提供する、Kubernetes Security Essentials から Security and the Linux Kernel までのサイバーセキュリティに特化したコースの受講を検討してください。私たちは、この取り組みを継続し、今後のイベントやイニシアチブに参加して、すべてのコミュニティのセキュリティを強化していくことを楽しみにしています。
